# Language specification
## Syntax
Используется `PEG` (Parsing Expression Grammar).
Лучше для практической реализации парсера в данном **статическом** компиляторе `Yummy`.
`PEG` обеспечивает однозначный парсинг без конфликтов благодаря упорядоченному выбору и встроенному lookahead, что упрощает обработку сложных конструкций вроде шаблонов или операторов с приоритетами.

- Неоднозначность: приоритетный выбор решается автоматически.
- Парсер: рекурсивный спуск (packrat), лексер опционален.
- Сложность языка: идеален для различных трюков `Rust`/`C++`.
- Скорость: **O(n)** с мемоизацией, проще отладка

PEG проще для прототипа и экспериментов с безопасными фичами, но все равно будет **основой базовой грамматики**.

## Ownership-model
Выбор пал на `borrow checker` (Rust-style) — он даёт максимальную защиту от **race conditions** и **use-after-free** без **runtime overhead**, идеально для **low-level** системного кода.

- Преимущества: **compile-time** гарантии, **zero-cost** абстракции, **lifetimes**.
- Недостатки: Сложный **checker**, **NLL** кривая обучения.

Но данное решение подходит для языка `Yummy`, который ориентируется на безопасность и не менее хорошую эффективность.
